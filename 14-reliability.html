精通阶段 第 14 篇

可靠性与看门狗

工业、医疗、车载等场景要求设备长期稳定运行。软件难免会死锁或跑飞，硬件与软件机制共同保障可靠性。

看门狗（Watchdog）

独立定时器，必须在超时前「喂狗」，否则复位 MCU。主循环或最高优先级任务定期喂狗；若某任务卡死导致无法喂狗，系统自动重启。

// 主循环或监控任务
while (1) {
    if (system_healthy()) {
        HAL_IWDG_Refresh(&hiwdg);
    }
    osDelay(500);
}

异常处理

  • 实现 HardFault_Handler，保存上下文、写日志、安全复位。
  • 断言与错误码：生产环境关闭 assert，改用可恢复的错误处理。
  • 双备份配置：Flash 中关键参数 A/B 区切换。

OTA 远程升级

  1. 下载固件到备份分区，校验 CRC / 签名。
  2. 切换 Boot 标志，重启进入新固件。
  3. 失败则回滚到旧版本（A/B 分区方案）。

现场稳定性

EMC 抗干扰、电源纹波、温湿度范围、长期老化测试，都是产品化不可或缺的一环。
