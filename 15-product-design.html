精通阶段 第 15 篇

项目实战：完整产品设计

从「会写驱动」到「做出产品」，还需要工程化思维。以下是一个物联网采集节点的设计范例。

1. 需求分析

  • 采集温湿度，每 5 分钟上报云端。
  • 电池供电，目标续航 1 年。
  • 支持 OTA，现场可维护。

2. 架构拆分

┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  Sensor     │────▶│  App Logic  │────▶│  Cloud API  │
│  Driver     │     │  (RTOS)     │     │  (MQTT)     │
└─────────────┘     └─────────────┘     └─────────────┘
       │                    │                    │
       └────────────────────┴────────────────────┘
                    Power Manager + WDT

3. 模块划分

  • BSP：板级支持，引脚、时钟、外设初始化。
  • Middleware：协议栈、文件系统、OTA。
  • Application：业务状态机，不直接操作寄存器。

4. 测试与量产

  1. 单元测试：驱动与算法模块。
  2. 集成测试：长时间跑机、高低温。
  3. 产测：ICT、烧录、射频校准、整机功能检。
  4. 文档：BOM、原理图、软件版本说明、维护手册。

5. 持续学习

嵌入式领域宽广：无线（BLE/LoRa/5G）、视觉、功能安全（ISO 26262）、RISC-V 等。保持阅读 datasheet、参考设计，并在真实项目中迭代。
