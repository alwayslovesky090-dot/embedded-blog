嵌入式 Linux 入门

当 MCU 算力不足、需要跑网络协议栈、数据库或复杂 UI 时，常选用 ARM + Linux 方案（如 i.MX、RK、全志、树莓派等）。

系统组成

  1. Bootloader：U-Boot 等，负责上电初始化、加载内核。
  2. Linux Kernel：进程调度、驱动框架、网络栈。
  3. Rootfs：根文件系统，含 busybox、库、应用程序。
  4. Device Tree：描述硬件拓扑，内核据此加载驱动。

交叉编译

在 PC（x86）上编译 ARM 程序，使用工具链如 aarch64-linux-gnu-gcc：

aarch64-linux-gnu-gcc -o hello hello.c
scp hello root@192.168.1.10:/tmp/

与裸机开发的差异

  • 应用运行在用户态，通过系统调用访问硬件。
  • 驱动在内核态，需遵循 Linux 驱动模型。
  • 调试手段：ssh、gdb、strace、dmesg 等更丰富。

学习路径

熟悉 Linux 命令行 → 交叉编译简单程序 → 理解设备树 → 阅读现有驱动 → 尝试字符设备驱动。
