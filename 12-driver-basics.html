驱动开发基础

Linux 驱动是应用与硬件之间的桥梁。入门从字符设备驱动开始最合适。

驱动类型

  • 字符设备：按字节流访问，如 GPIO、UART。
  • 块设备：按块访问，如 eMMC、SD 卡。
  • 网络设备：以太网、Wi-Fi 网卡。

字符设备框架

static struct file_operations fops = {
    .owner = THIS_MODULE,
    .open  = my_open,
    .read  = my_read,
    .write = my_write,
    .release = my_release,
};

static int __init my_init(void) {
    alloc_chrdev_region(&devno, 0, 1, "mydev");
    cdev_init(&my_cdev, &fops);
    cdev_add(&my_cdev, devno, 1);
    return 0;
}

Platform 驱动与设备树

现代 ARM 平台通过设备树描述硬件，驱动用 of_match_table 与节点匹配：

// 设备树片段
myled {
    compatible = "vendor,my-led";
    gpios = <&gpio 18 GPIO_ACTIVE_HIGH>;
};

用户态访问

应用通过 /dev/mydev 或 sysfs（/sys/class/...）与驱动交互，避免直接操作寄存器。
