C 语言嵌入式要点

嵌入式开发几乎离不开 C 语言。除了语法基础，还需要掌握与硬件紧密相关的写法。

位操作

寄存器配置常通过置位、清位完成：

// 将第 5 位置 1
REG |= (1 << 5);

// 将第 3 位清 0
REG &= ~(1 << 3);

// 读取第 2 位
if (REG & (1 << 2)) { /* ... */ }

volatile 关键字

寄存器、中断共享变量可能被硬件或中断意外修改，编译器不应优化掉对它的访问：

volatile uint32_t *const GPIO_ODR = (uint32_t *)0x4001080C;

指针与内存映射

外设寄存器在芯片手册中有固定地址，通过指针直接访问：

#define PERIPH_BASE  0x40000000UL
#define GPIOA_BASE   (PERIPH_BASE + 0x10800UL)
#define GPIOA_CRL    (*(volatile uint32_t *)(GPIOA_BASE + 0x00))

固定宽度整数类型

使用 stdint.h 中的 uint8_t、uint16_t、uint32_t，避免 int 在不同平台上长度不一致。

栈与堆

  • MCU RAM 很小，避免大数组、递归、频繁 malloc。
  • 中断服务函数（ISR）应尽量短，不要在 ISR 里调用可能阻塞的 API。
建议：阅读芯片 Reference Manual 中的寄存器章节，对照代码理解每一位的含义。
