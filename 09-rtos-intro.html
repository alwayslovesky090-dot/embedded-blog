RTOS 入门

当项目同时要做通信、采集、UI、控制时，单线程 while(1) 难以维护。RTOS（实时操作系统）通过多任务调度解决并发问题。

核心概念

  • 任务（Task）：独立线程，各自有栈空间。
  • 调度器：按优先级或时间片切换任务。
  • 队列（Queue）：任务间安全传递数据。
  • 信号量 / 互斥量：同步与保护共享资源。

FreeRTOS 最小示例

void led_task(void *arg) {
    while (1) {
        led_toggle();
        vTaskDelay(pdMS_TO_TICKS(500));
    }
}

int main(void) {
    // 硬件初始化 ...
    xTaskCreate(led_task, "LED", 128, NULL, 1, NULL);
    vTaskStartScheduler();
    while (1);
}

何时需要 RTOS

  • 多个逻辑并行、有阻塞等待（等网络、等传感器）。
  • 需要严格优先级（控制环高于日志任务）。

简单单功能项目可继续裸机；复杂度上升再引入 RTOS，避免过早抽象。
