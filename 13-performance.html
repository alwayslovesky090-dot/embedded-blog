精通阶段 第 13 篇

性能优化与调试

产品级嵌入式不仅要「能跑」，还要在有限资源下稳定、高效。优化前先测量，避免盲目改代码。

常见瓶颈

  • CPU 占用过高：算法复杂度、频繁中断、轮询等待。
  • 内存不足：栈溢出、内存泄漏、碎片（若用堆）。
  • I/O 阻塞：同步读写 Flash、无 DMA 的 SPI 传输。

DMA（直接内存访问）

外设与 RAM 之间传数据不经过 CPU 逐字节搬运，适合大批量 ADC、音频、显示缓冲传输，显著降低 CPU 负载。

Cache 与一致性

带 Cache 的 MPU 上，DMA 缓冲区需考虑 Cache 行对齐与 invalidate/clean，否则可能读到脏数据。

调试工具

  • 逻辑分析仪 / 示波器：总线时序、中断间隔。
  • SWO / ITM：低开销跟踪打印。
  • gprof / perf（Linux）：热点函数分析。
  • HardFault 分析：查看 LR/PC 栈回溯定位崩溃点。
原则：先 profile 再优化；优先改算法与架构，最后再抠微优化。
