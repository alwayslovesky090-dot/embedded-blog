入门阶段 第 02 篇

开发环境搭建

开始嵌入式开发前，需要准备硬件、软件工具链和调试手段。下面以 STM32 为例说明通用流程，其他平台思路类似。

硬件准备

  • 开发板：入门推荐 STM32F103「蓝 pill」或 Nucleo 系列，资料丰富。
  • 调试器：板载 ST-Link 或外接 J-Link；ESP32 通常用 USB 直连。
  • 杜邦线、面包板、USB 线：用于外接 LED、传感器等。

软件工具

1. IDE / 编辑器

  • STM32CubeIDE：ST 官方免费 IDE，集成编译与调试。
  • VS Code + 插件：PlatformIO 或 Cortex-Debug，轻量灵活。
  • Keil / IAR：商业 IDE，部分芯片厂商提供有限免费版。

2. 工具链

ARM GCC 交叉编译器（arm-none-eabi-gcc）将 C 代码编译成 MCU 可执行的机器码。CubeIDE 和 PlatformIO 通常已内置。

3. 配置与代码生成

STM32CubeMX 可图形化配置时钟、引脚、外设，并生成初始化代码，大幅降低上手难度。

第一次连接验证

  1. 安装 IDE 与驱动（ST-Link 驱动等）。
  2. 用 USB 连接开发板，确认系统识别到调试器。
  3. 导入或创建一个 Empty 工程，编译并下载到板子。
  4. 若能进入调试、单步执行，说明环境 OK。
提示：遇到「无法识别设备」时，优先检查 USB 线是否支持数据传输、驱动是否安装、开发板跳线是否处于调试模式。
