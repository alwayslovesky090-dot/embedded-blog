中断与定时器

轮询（不断查询 GPIO）浪费 CPU。中断（Interrupt）让外设事件触发 CPU 跳转执行 ISR，定时器则提供精确的时间基准。

中断基本概念

  • 中断源：按键、串口收到数据、定时器溢出等。
  • ISR：Interrupt Service Routine，中断服务函数，应尽量短小。
  • NVIC：嵌套向量中断控制器，管理优先级与嵌套。

优先级

高优先级中断可打断低优先级。合理设置优先级可避免串口丢数据或控制环路抖动。

定时器用途

  1. 周期性任务（如每 1ms 扫描按键）。
  2. PWM 输出（电机、LED 调光）。
  3. 输入捕获（测量脉冲宽度、频率）。
  4. 软件延时替代 HAL_Delay 阻塞。

定时器中断示例思路

// TIM2 每 1ms 中断一次
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim) {
    if (htim->Instance == TIM2) {
        tick_ms++;
        if (tick_ms >= 1000) {
            tick_ms = 0;
            flag_1s = 1;  // 主循环处理耗时逻辑
        }
    }
}
注意：ISR 中不要调用 HAL_Delay、printf（若阻塞）等耗时操作；复杂逻辑用标志位通知主循环处理。
